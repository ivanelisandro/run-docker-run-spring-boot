Description

Now that you have created a Dockerfile for your application, it's time to build a custom Docker image based on it. This will allow you to package your application into a portable container image that can be run anywhere.

In this stage, you will use Docker to build an image from your Dockerfile. The image should be tagged as hyper-web-app:latest so it can be easily identified and used in subsequent stages.

By completing this stage, you will gain hands-on experience in building Docker images and tagging them for use in containerized environments.

Objectives

Here are the objectives for this stage:

  • Use the appropriate Docker command to build the image from your Dockerfile.

  • Tag the image as hyper-web-app:latest.
